# Shipley Global Knowledge References

These files are commit-safe global references for shaping Ariadne Thread's capture methodology, domain language, decision gates, and artifact workflows.

## Files

- `Shipley Capture Guide.pdf`: Capture lifecycle, strategy, customer focus, gate discipline, and pursuit-management guidance.
- `Shipley Proposal Guide.pdf`: Proposal planning, compliance, writing, review, and production guidance.
- `shipley_extracted.json`: Extracted machine-readable reference content for future knowledge workflows.

## Usage Rules

- Treat these as global knowledge references, not application runtime code.
- Do not build a RAG/indexing pipeline until architecture work chooses the knowledge engine and storage shape.
- Use these references to keep terminology, workflows, and generated artifacts aligned with Shipley principles.
